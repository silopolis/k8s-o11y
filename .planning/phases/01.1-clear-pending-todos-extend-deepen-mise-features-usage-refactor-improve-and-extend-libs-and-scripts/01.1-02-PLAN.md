## Objective

Implement comprehensive quality gates using pre-commit hooks with markdownlint-cli2, shellcheck, and supporting checks. Establish automated code quality enforcement that runs on every commit with auto-fix capabilities where possible.

## Context

**Locked Decisions (NON-NEGOTIABLE):**

1. **Tool Installation:** markdownlint-cli2 and shellcheck must be installed via mise BEFORE pre-commit hooks are configured
2. **Pre-commit Strictness:** Mixed strictness - some checks fail/block, some warn only
3. **Check Set:** markdownlint + shellcheck + YAML validation + JSON validation + secret detection + trailing whitespace + end-of-file
4. **Auto-fix Strategy:** Auto-fix what can be fixed, fail commit on remaining unfixable issues
5. **Scope:** Staged files only (not entire repo)
6. **Shellcheck for Libs:** SC2034 must be disabled for lib/ files (variables sourced elsewhere, not used within file)

**Dependencies:**
- **Depends on Plan 01.1-01** - mise infrastructure must be ready with tool installation capability
- Tools (markdownlint-cli2, shellcheck) must be in mise.toml before pre-commit install

**Current State:**
- No pre-commit configuration exists
- markdownlint-cli2.yaml exists but isn't integrated with pre-commit
- No shellcheck configuration exists
- mise.toml has lint_md and lint_md_fix tasks but not as file-based tasks

<task type="auto" id="4">
  <title>Create .pre-commit-config.yaml</title>
  <files>.pre-commit-config.yaml</files>
  <action>Create `.pre-commit-config.yaml` with the following hooks configured for MIXED strictness (some fail, some warn):

  ```yaml
  # Pre-commit hooks configuration
  # Run: pre-commit install (one-time setup)
  # Auto-fix: pre-commit run --all-files
  
  repos:
    # Trailing whitespace and end-of-file fixes (auto-fix, warn-only after fix)
    - repo: https://github.com/pre-commit/pre-commit-hooks
      rev: v4.5.0
      hooks:
        - id: trailing-whitespace
          stages: [pre-commit]
        - id: end-of-file-fixer
          stages: [pre-commit]
        - id: check-yaml
          stages: [pre-commit]
          args: [--allow-multiple-documents]
        - id: check-json
          stages: [pre-commit]
        - id: detect-private-key
          stages: [pre-commit]
          severity: error  # FAIL - security critical
    
    # Markdown linting (fail on errors)
    - repo: https://github.com/DavidAnson/markdownlint-cli2
      rev: v0.14.0
      hooks:
        - id: markdownlint-cli2
          stages: [pre-commit]
          args: [--fix]  # Auto-fix what can be fixed
          pass_filenames: true
          files: \.md$
    
    # Shellcheck (fail on errors, but with SC2034 disabled for libs)
    - repo: https://github.com/shellcheck-py/shellcheck-py
      rev: v0.10.0.1
      hooks:
        - id: shellcheck
          stages: [pre-commit]
          args: [-x]  # Follow sourced files
          files: \.sh$
          exclude: ^\.mise/tasks/  # Exclude if they cause issues
  ```

  Key points:
  - detect-private-key FAILS (security critical)
  - markdownlint-cli2 and shellcheck FAIL but auto-fix first
  - YAML/JSON validation WARN on issues
  - All hooks use stages: [pre-commit] for staged-files-only behavior</action>
  <verify>Run `pre-commit validate-config` to verify the configuration is valid</verify>
  <done>.pre-commit-config.yaml exists with all required hooks and proper fail/warn configuration</done>
</task>
