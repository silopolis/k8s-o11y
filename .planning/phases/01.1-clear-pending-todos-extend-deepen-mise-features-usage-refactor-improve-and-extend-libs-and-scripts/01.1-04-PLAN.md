## Objective

Fix Talos control plane metrics exposition and configure Prometheus scraping. Enable monitoring of control plane components (kube-apiserver, kube-scheduler, kube-controller-manager, etcd) that run as static pods in Talos Linux.

## Context

**Locked Decisions (NON-NEGOTIABLE):**

1. **Critical Blocker:** This TODO is a critical blocker that must be resolved before adding new capabilities
2. **Implementation Approach:** Use `extraArgs` in Talos machine config to expose metrics endpoints
3. **Prometheus Scraping:** ServiceMonitor for control plane components
4. **Recording Rules:** Follow colon-separated naming convention: `app:metric:aggregation`
5. **Alert Names:** Use severity prefix: `WARN`, `ERR`, or `CRIT`

**Dependencies:**
- **Depends on Plan 01.1-01** - mise tasks directory structure must exist for organizing Talos-specific tasks
- Can run in parallel with Wave 3 (Documentation) since they target different concerns
- Requires working Talos cluster and deployed kube-prometheus-stack

**Current State:**
- scripts/apply-talos-metrics-config.sh exists (from pending TODO research)
- Talos control plane metrics are NOT currently exposed
- No ServiceMonitor for Talos control plane components
- No recording rules for Talos-specific metrics
- Control plane pods run as static pods (different from standard Kubernetes)

**Talos-Specific Considerations:**
- Talos runs control plane components as static pods managed by the OS
- Metrics endpoints must be explicitly exposed via machine configuration
- Talos API is used to apply configuration changes
- Control plane nodes need specific `extraArgs` for each component

  ```yaml
  kind: ServiceMonitor
  ```

<task type="auto" id="3">
  <title>Create Talos control plane recording rules</title>
  <files>config/prometheus/talos-control-plane-rules.yaml</files>
  <action>Create `config/prometheus/talos-control-plane-rules.yaml` with recording rules and alerts for Talos control plane metrics:

  ```yaml
  apiVersion: monitoring.coreos.com/v1
  kind: PrometheusRule
  metadata:
    name: talos-control-plane
    namespace: monitoring
    labels:
      app.kubernetes.io/name: talos-control-plane
      app.kubernetes.io/part-of: kube-prometheus-stack
  spec:
    groups:
      - name: talos-control-plane.recording
        interval: 30s
        rules:
          # API server request rate
          - record: talos:apiserver:request_rate
            expr: sum(rate(apiserver_request_total[5m]))
          
          # API server error rate
          - record: talos:apiserver:error_rate
            expr: sum(rate(apiserver_request_total{code=~"5.."}[5m]))
          
          # Scheduler schedule attempts
          - record: talos:scheduler:schedule_attempts
            expr: sum(scheduler_schedule_attempts_total)
          
          # Controller manager workqueue depth
          - record: talos:controller:workqueue_depth
            expr: avg(workqueue_depth)
          
          # Etcd leader changes
          - record: talos:etcd:leader_changes
            expr: sum(etcd_server_leader_changes_seen_total)
      
      - name: talos-control-plane.alerts
        interval: 30s
        rules:
          # WARN: High API server error rate
          - alert: WARN_TalosAPIServerHighErrorRate
            expr: talos:apiserver:error_rate / talos:apiserver:request_rate > 0.05
            for: 5m
            labels:
              severity: warning
            annotations:
              summary: "Talos API server error rate is high"
              description: "API server error rate is {{ $value | humanizePercentage }} over the last 5m"
          
          # ERR: etcd leader changes
          - alert: ERR_TalosEtcdLeaderChanges
            expr: talos:etcd:leader_changes > 0
            for: 1m
            labels:
              severity: error
            annotations:
              summary: "Talos etcd has seen leader changes"
              description: "etcd leader has changed {{ $value }} times recently"
          
          # WARN: Controller manager workqueue backing up
          - alert: WARN_TalosControllerWorkqueueDepth
            expr: talos:controller:workqueue_depth > 100
            for: 10m
            labels:
              severity: warning
            annotations:
              summary: "Talos controller workqueue is backing up"
              description: "Controller workqueue depth is {{ $value }}"
  ```

  Follow naming conventions:
  - Recording rules: `talos:component:metric` format
  - Alerts: `WARN_`, `ERR_`, or `CRIT_` prefix with descriptive names
  - Include `for` duration for all alerts to avoid flapping
  - Provide summary and description annotations</action>
  <verify>Run `kubectl apply --dry-run=client -f config/prometheus/talos-control-plane-rules.yaml` to validate YAML; verify rules follow naming conventions</verify>

  ```bash
  #!/usr/bin/env bash
  # Verify Talos control plane metrics are being collected by Prometheus
  
  set -euo pipefail
  
  SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
  PROJECT_ROOT="$(cd "${SCRIPT_DIR}/../../.." && pwd)"
  
  source "${PROJECT_ROOT}/lib/output.sh"
  
  header "Verifying Talos Control Plane Metrics"
  
  # Get operator pod for executing queries
  OPERATOR_POD=$(kubectl get pods -n monitoring -l app.kubernetes.io/name=kube-prometheus-stack-prometheus-operator -o jsonpath='{.items[0].metadata.name}' 2>/dev/null || true)
  
  if [ -z "$OPERATOR_POD" ]; then
      error "Could not find prometheus-operator pod"
      exit 1
  fi
  
  echo "Checking control plane metrics availability..."
  echo ""
  
  # Check apiserver metrics
  echo "1. API Server metrics:"
  APISERVER_METRICS=$(kubectl exec -n monitoring "$OPERATOR_POD" -- \
      wget -qO- "http://kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090/api/v1/query?query=up{job=~\".*apiserver.*\"}" 2>/dev/null | \
      jq -r '.data.result | length' 2>/dev/null || echo "0")
  if [ "$APISERVER_METRICS" -gt 0 ]; then
      echo "   ✓ API server metrics available ($APISERVER_METRICS targets)"
  else
      echo "   ✗ API server metrics not found"
  fi
  echo ""
  
  # Check scheduler metrics
  echo "2. Scheduler metrics:"
  SCHEDULER_METRICS=$(kubectl exec -n monitoring "$OPERATOR_POD" -- \
      wget -qO- "http://kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090/api/v1/query?query=up{job=~\".*scheduler.*\"}" 2>/dev/null | \
      jq -r '.data.result | length' 2>/dev/null || echo "0")
  if [ "$SCHEDULER_METRICS" -gt 0 ]; then
      echo "   ✓ Scheduler metrics available ($SCHEDULER_METRICS targets)"
  else
      echo "   ✗ Scheduler metrics not found"
  fi
  echo ""
  
  # Check controller-manager metrics
  echo "3. Controller Manager metrics:"
  CONTROLLER_METRICS=$(kubectl exec -n monitoring "$OPERATOR_POD" -- \
      wget -qO- "http://kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090/api/v1/query?query=up{job=~\".*controller.*\"}" 2>/dev/null | \
      jq -r '.data.result | length' 2>/dev/null || echo "0")
  if [ "$CONTROLLER_METRICS" -gt 0 ]; then
      echo "   ✓ Controller manager metrics available ($CONTROLLER_METRICS targets)"
  else
      echo "   ✗ Controller manager metrics not found"
  fi
  echo ""
  
  # Check etcd metrics
  echo "4. etcd metrics:"
  ETCD_METRICS=$(kubectl exec -n monitoring "$OPERATOR_POD" -- \
      wget -qO- "http://kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090/api/v1/query?query=up{job=~\".*etcd.*\"}" 2>/dev/null | \
      jq -r '.data.result | length' 2>/dev/null || echo "0")
  if [ "$ETCD_METRICS" -gt 0 ]; then
      echo "   ✓ etcd metrics available ($ETCD_METRICS targets)"
  else
      echo "   ✗ etcd metrics not found"
  fi
  echo ""
  
  # Check recording rules
  echo "5. Recording rules:"
  RECORDING_RULES=$(kubectl exec -n monitoring "$OPERATOR_POD" -- \
      wget -qO- "http://kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090/api/v1/rules" 2>/dev/null | \
      jq -r '.data.groups[] | select(.name | contains("talos")) | .rules | length' 2>/dev/null | awk '{sum+=$1} END {print sum}' || echo "0")
  if [ "$RECORDING_RULES" -gt 0 ]; then
      echo "   ✓ Recording rules loaded ($RECORDING_RULES rules)"
  else
      echo "   ✗ Recording rules not found"
  fi
  echo ""
  
  header "Verification Summary"
  TOTAL_TARGETS=$((APISERVER_METRICS + SCHEDULER_METRICS + CONTROLLER_METRICS + ETCD_METRICS))
  if [ "$TOTAL_TARGETS" -ge 4 ]; then
      success "Control plane metrics are being collected from all components"
  else
      warning "Some control plane metrics may not be available yet. Wait a few minutes and retry."
  fi
  ```
