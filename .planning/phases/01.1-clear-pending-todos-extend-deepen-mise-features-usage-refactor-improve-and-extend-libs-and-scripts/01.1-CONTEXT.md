## Implementation Decisions


### TODO Execution Order

- Execute TODOs in **waves** (not all at once)
- **Infrastructure first** - mise secrets, tasks directory, tools
- **Quality gates second** - pre-commit, markdownlint, shellcheck (depends on tools being installed first)
- **Documentation/utilities third** - README, getter functions
- **Fix existing issues before adding new capabilities** - PATH issues, Talos metrics fixes take priority


### Pre-commit Dependencies

- Pre-commit hooks **depend on tools being installed first**
- Install markdownlint-cli2 and shellcheck via mise before configuring hooks
- Tools and hooks can be in same wave, but tools must be available before hook configuration runs


### Critical Blockers

- Fix mise/brew PATH assumptions in preflight script (existing issue)
- Fix Talos control plane metrics exposition (existing issue)
- These must be resolved before adding new capabilities


### Secrets Management Backend

- **SOPS** (Mozilla SOPS) selected as the backend
- **All secrets** - both external service credentials (API keys, tokens) AND infrastructure secrets (cluster, kubeconfig, Talos)
- **Mixed access pattern** - some via env vars, some file-based, some CLI direct (depends on tool requirements)
- **Hybrid sharing** - infrastructure secrets shared across team, personal API keys remain individual


### Pre-commit Strictness (Updated)

- **Mixed strictness** - some checks fail/block, some warn only (to be defined per check)
- **Extended check set** - markdownlint + shellcheck + YAML validation + JSON validation + secret detection + trailing whitespace + end-of-file
- **Auto-fix then fail** - auto-fix what can be fixed, fail commit on remaining unfixable issues
- **Staged files only** - only check files being committed (not entire repo)


### Mise Tasks Organization (New)

- **Functional and resource groups** in `.mise/tasks/` - deploy/, check/, lint/, utils/, cluster/, secrets/
- **Hierarchical naming** within groups - e.g., `deploy/stack`, `lint/helmfile`, `verify/phase1`, `cluster/context`
- **Full migration** - move all inline tasks in mise.toml to files
- **Category prefix invocation** - `mise run deploy:stack`, `mise run lint:all`, `mise run verify:phase1`


### Script/Libs Structure (New)

- **Top-level directories** - `.mise/tasks/` for task scripts, `lib/` for shared libraries (at repo root)
- **Both types of libs** - generic utilities (logging, error handling) AND domain-specific helpers (k8s, monitoring, deployment)
- **Thin wrapper scripts** - scripts call lib functions, contain minimal logic themselves
- **All .sh extension** - both libs and scripts use .sh extension, tasks are executable


### Claude's Discretion

- Exact wave grouping within categories
