  <title>Generate age encryption key and configure SOPS</title>
  <files>~/.config/mise/age.txt, .sops.yaml</files>
  <action>
    Generate an age key pair for SOPS encryption. This is a ONE-TIME setup that must be completed before any secrets can be encrypted.
    
    Steps:
    1. Create directory: `mkdir -p ~/.config/mise/`
    2. Generate key: `age-keygen -o ~/.config/mise/age.txt`
    3. Capture the public key (starts with 'age1') from the output
    4. Create `.sops.yaml` in project root:
       ```yaml
       creation_rules:
         - path_regex: \.env\.json$
           age: <public_key_from_step_3>
       ```
    
    IMPORTANT: The private key in ~/.config/mise/age.txt is CRITICAL. If lost, all secrets are unrecoverable. Per re-research insights, a backup strategy should be documented.
  </action>
  <verify>Run `cat ~/.config/mise/age.txt` and verify private key exists; run `cat .sops.yaml` and verify public key is configured</verify>
  <done>age key exists at ~/.config/mise/age.txt and .sops.yaml references the public key</done>
</task>
