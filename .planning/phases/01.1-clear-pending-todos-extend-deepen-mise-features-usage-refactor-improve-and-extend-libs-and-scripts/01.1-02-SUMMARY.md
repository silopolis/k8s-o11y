# Phase 01.1 Plan 02 Summary: Quality Gates - Pre-commit, Linting, and Hooks


## Overview

Established quality gates by configuring pre-commit hooks that use the mise exec pattern for version consistency, setting up markdownlint and shellcheck configurations, and creating reusable linting infrastructure.


## Changes Made


### 1. Pre-commit Configuration (`.pre-commit-config.yaml`)

Created pre-commit configuration using the mise exec pattern for version consistency:

- **Trailing whitespace hook**: Auto-fixes trailing whitespace issues
- **End of file fixer**: Ensures files end with a single newline
- **YAML validation**: Validates YAML syntax
- **JSON validation**: Validates JSON syntax
- **Markdown linting**: Uses `mise exec -- markdownlint-cli2` for markdown linting
- **Shellcheck**: Uses `mise exec -- shellcheck` for shell script linting

Key design decisions:
- Uses `mise exec --` pattern instead of pre-commit installing its own versions
- Ensures consistency between `mise run lint:*` and pre-commit hooks
- Mixed strictness: shellcheck blocks commits, markdownlint warns


### 2. Markdownlint Configuration (`.markdownlint-cli2.yaml`)

Existing configuration enhanced with project-specific rules:

- **MD013**: Disabled (semantic line breaks preferred over hard line limits)
- **MD033**: Disabled (allows inline HTML for MyST/Quarto)
- **MD041**: Disabled (front matter `title:` satisfies first-line-h1)
- **MD049/MD050**: Enforces asterisk-style emphasis (`*italic*` and `**bold**`)
- **MD029**: Uses `one_or_ordered` style for lists
- **MD022**: Enforces blank lines around headings
- **MD026**: Restricts trailing punctuation in headings


### 3. Shared Linting Library (`lib/lint.sh`)

Created reusable linting functions:

- `lint_markdown()`: Run markdownlint on specified files
- `lint_markdown_fix()`: Run markdownlint with auto-fix
- `lint_shell()`: Run shellcheck on shell scripts
- `lint_shell_strict()`: Run shellcheck with warning severity
- `lint_check_tools()`: Verify all linting tools are available


### 4. Mise Lint Tasks

Created file-based lint tasks in `.mise/tasks/lint/`:

- **`lint:markdown`**: Points to `.mise/tasks/lint/markdown/` directory
- **`lint:shell`**: Points to `.mise/tasks/lint/shell/` directory
  - `shell/all`: Runs shellcheck on all shell scripts
  - `shell/_default`: Default shell lint task
- **`lint:all`**: Aggregate task that runs both markdown and shell linting

Updated `mise.toml` with task references:
```toml
[tasks."lint:markdown"]
file = ".mise/tasks/lint/markdown"

[tasks."lint:shell"]
file = ".mise/tasks/lint/shell"
```

## Usage

**Run markdown linting:**
```bash
mise run lint:markdown
```

**Run shell linting:**
```bash
mise run lint:shell
```

**Run all linting:**
```bash
mise run lint:all
```

**Run pre-commit on staged files:**
```bash
mise exec -- pre-commit run
```

**Run pre-commit on all files:**
```bash
mise exec -- pre-commit run --all-files
```

**Verify Wave 2 setup:**
```bash
mise run verify:wave2
```
