## Objective

Establish infrastructure foundation by fixing critical PATH assumptions, setting up mise secrets management with SOPS, and refactoring mise tasks into a hierarchical directory structure. These are the foundational capabilities that all other TODOs depend on.

## Context

**Locked Decisions (NON-NEGOTIABLE):**

1. **Secrets Backend:** SOPS with age encryption
2. **Task Organization:** Functional groups in `.mise/tasks/` with hierarchical naming (deploy/stack, verify/phase1, lint/all)
3. **PATH Fix:** Remove mise/brew PATH assumptions from preflight.sh - this is a critical blocker
4. **Tool Installation:** sops and age must be available via mise before any secrets operations

**Dependencies:**
- This is Wave 1 - no dependencies on other TODOs
- Wave 2 (Quality Gates) depends on this plan's tool installations
- All other plans depend on the infrastructure foundation established here

**Current State:**
- preflight.sh has hardcoded PATH assumptions for mise/brew (lines 17-21)
- mise.toml has all tasks defined inline (complex and simple mixed)
- No secrets management configured
- No SOPS/age tools in mise.toml

<task type="checkpoint:human-action" id="3">
  <title>Generate age encryption key</title>
  <files>~/.config/mise/age.txt, .sops.yaml</files>
  <action>Generate an age key pair for SOPS encryption. Run `age-keygen -o ~/.config/mise/age.txt` and capture the public key output. Create `.sops.yaml` file in project root with the following content:
  ```yaml
  creation_rules:
    - path_regex: \.env\.json$
      age: <public_key>
  ```
  The public key will be displayed when running age-keygen (starts with 'age1'). This key is required for all subsequent secrets operations.</action>
  <verify>Run `cat ~/.config/mise/age.txt` and verify private key exists; run `cat .sops.yaml` and verify public key is configured</verify>
  <done>age key exists at ~/.config/mise/age.txt and .sops.yaml references the public key</done>
</task>

<task type="auto" id="4">
  <title>Create encrypted secrets file</title>
  <files>.env.json</files>
  <action>Create an encrypted `.env.json` file for storing secrets. Start with a placeholder structure:
  ```json
  {
    "GRAFANA_ADMIN_PASSWORD": "placeholder_to_be_replaced",
    "TALOS_ADMIN_PASSWORD": "placeholder_to_be_replaced"
  }
  ```
  Then encrypt it with: `sops encrypt -i .env.json`. Ensure the file is encrypted (should contain 'sops' metadata after encryption). Add `.env.json` to .gitignore to prevent accidental commits of decrypted content, but note that encrypted SOPS files ARE safe to commit.</action>
  <verify>Run `sops decrypt .env.json` and verify it shows the decrypted JSON; verify file contains SOPS metadata indicating it's encrypted</verify>
  <done>.env.json exists and is SOPS-encrypted with valid JSON structure</done>
</task>

<task type="auto" id="5">

</task>
