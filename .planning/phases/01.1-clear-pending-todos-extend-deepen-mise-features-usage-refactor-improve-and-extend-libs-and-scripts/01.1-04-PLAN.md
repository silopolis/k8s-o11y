    ```bash
    #!/usr/bin/env bash
    # Fix script for Talos control plane metrics issues
    # Usage: ./scripts/fix-control-plane-metrics.sh [--apply]

    set -euo pipefail

    SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    cd "${SCRIPT_DIR}/.."

    source lib/common.sh
    source lib/kubernetes.sh

    APPLY=false
    [[ "${1:-}" == "--apply" ]] && APPLY=true

    info "Control Plane Metrics Fix Script"
    echo

    # Check current state
    info "Checking current control plane metrics status..."

    # Check if controller-manager metrics endpoint is accessible
    local cm_status="unknown"
    if kubectl get --raw /api/v1/nodes/$(kubectl get nodes -l node-role.kubernetes.io/control-plane -o jsonpath='{.items[0].metadata.name}'):10257/proxy/metrics 2>/dev/null | head -1 | grep -q "#"; then
      cm_status="accessible"
      success "Controller-manager metrics endpoint accessible"
    else
      cm_status="not-accessible"
      warn "Controller-manager metrics endpoint not accessible"
    fi

    # Check if scheduler metrics endpoint is accessible
    local sched_status="unknown"
    if kubectl get --raw /api/v1/nodes/$(kubectl get nodes -l node-role.kubernetes.io/control-plane -o jsonpath='{.items[0].metadata.name}'):10259/proxy/metrics 2>/dev/null | head -1 | grep -q "#"; then
      sched_status="accessible"
      success "Scheduler metrics endpoint accessible"
    else
      sched_status="not-accessible"
      warn "Scheduler metrics endpoint not accessible"
    fi

    # Check if ServiceMonitors exist
    info "Checking ServiceMonitors..."
    kubectl get servicemonitor -n monitoring control-plane-metrics kube-scheduler-metrics kube-proxy-metrics 2>/dev/null && \
      success "ServiceMonitors exist" || \
      warn "ServiceMonitors not found"

    echo
    info "Status Summary:"
    echo "  Controller-manager: $cm_status"
    echo "  Scheduler: $sched_status"
    echo "  etcd: NOT CONFIGURED (requires mTLS, see Phase 01.2)"
    echo

    if [[ "$APPLY" == true ]]; then
      info "Applying fixes..."
      mise run deploy:control-plane-monitoring
      success "Fixes applied"
    else
      info "Run with --apply to apply fixes"
    fi
    ```

    #!/usr/bin/env bash
    # [MISE] description="Verify control plane metrics are being scraped"
    # [MISE] depends=[]

    set -euo pipefail

    source lib/common.sh

    info "Verifying control plane metrics..."

    # Check ServiceMonitors exist
    info "Checking ServiceMonitors..."
    kubectl get servicemonitor -n monitoring control-plane-metrics >/dev/null 2>&1 || \
      die "control-plane-metrics ServiceMonitor not found"
    kubectl get servicemonitor -n monitoring kube-scheduler-metrics >/dev/null 2>&1 || \

      die "kube-proxy-metrics ServiceMonitor not found"
    success "All ServiceMonitors exist"

    # Check Prometheus targets are up (if Prometheus is running)
    info "Checking Prometheus targets..."
    local prom_url="http://localhost:9090"
    if kubectl port-forward -n monitoring svc/prometheus-kube-prometheus-stack-prometheus 9090:9090 --address 0.0.0.0 &>/dev/null &
    then
      sleep 2
      curl -s "$prom_url/api/v1/targets" | jq -r '.data.activeTargets[] | select(.labels.job | contains("kube-controller-manager","kube-scheduler","kube-proxy")) | {job: .labels.job, health: .health}' 2>/dev/null || \
        warn "Could not query Prometheus targets (may not be running yet)"
      pkill -f "port-forward.*9090" 2>/dev/null || true
    else
      warn "Could not port-forward to Prometheus (may not be running)"
    fi

    # Check that metrics endpoints are at least responding
    info "Checking metrics endpoints..."
    # Try to access via node proxy
    local control_plane_node
    control_plane_node=$(kubectl get nodes -l node-role.kubernetes.io/control-plane -o jsonpath='{.items[0].metadata.name}' 2>/dev/null) || true

    if [[ -n "$control_plane_node" ]]; then
      # Note: These endpoints may require authentication
      kubectl get --raw "/api/v1/nodes/$control_plane_node:10257/proxy/metrics" 2>/dev/null | head -5 || \
        warn "Controller-manager metrics endpoint not directly accessible (may need ServiceMonitor)"
    fi

    success "Control plane verification complete"
    info "Note: etcd metrics verification deferred to Phase 01.2 (mTLS)"
    ```

    # Control Plane Monitoring

    Configuration for monitoring Kubernetes control plane components via Prometheus.

    ## Overview

    This setup monitors the following control plane components:

    - **kube-controller-manager** - Controller management metrics (port 10257)
    - **kube-scheduler** - Scheduling metrics (port 10259)
    - **kube-proxy** - Network proxy metrics (various HTTP ports)

    ## Excluded: etcd

    **etcd metrics are NOT included** in this configuration due to mTLS requirements.

    etcd exposes metrics on port 2381, but requires client certificates for authentication.
    Unlike other control plane components, etcd uses mutual TLS (mTLS) which requires:

    1. Valid client certificates
    2. Certificate rotation handling
    3. Proper certificate distribution to Prometheus

    etcd monitoring will be addressed in **Phase 01.2** with a proper certificate management solution.

    ## Configuration

    ### Talos Machine Config Patch

    The file `.talos/patches/control-plane-metrics.yaml` contains the machine configuration patch
    that enables metrics endpoints on control plane nodes.

    To apply:
    ```bash
    talosctl patch --patch-file .talos/patches/control-plane-metrics.yaml
    ```

    ### ServiceMonitors

    ServiceMonitors are defined in `manifests/prometheus/servicemonitor-control-plane.yaml`:

    - `control-plane-metrics` - Scrapes controller-manager
    - `kube-scheduler-metrics` - Scrapes scheduler
    - `kube-proxy-metrics` - Scrapes kube-proxy on all nodes

    To deploy:
    ```bash
    kubectl apply -f manifests/prometheus/servicemonitor-control-plane.yaml
    ```

    ## Metrics Available

    ### Controller Manager

    - `workqueue_depth` - Current depth of workqueue
    - `workqueue_adds_total` - Total workqueue adds
    - `node_collector_*` - Node-related metrics
    - `certificate_*` - Certificate expiration metrics

    ### Scheduler

    - `scheduler_*` - Scheduling latency, attempts, failures
    - `apiserver_*` - API server client metrics

    ### Kube-proxy

    - `kubeproxy_*` - Network proxy metrics
    - `rest_client_*` - API client metrics

    ## Verification

    Check that metrics are being scraped:

    ```bash
    # Via mise task
    mise run verify:control-plane

    # Manual check
    kubectl get servicemonitor -n monitoring
    ```

    ## Troubleshooting

    ### "Connection refused" on metrics endpoints

    The Talos patch may not have been applied. Check:
    ```bash
    # Check if patch is applied
    talosctl get machineconfig
    ```

    ### ServiceMonitors not found

    Apply the manifests:
    ```bash
    kubectl apply -f manifests/prometheus/servicemonitor-control-plane.yaml
    ```

    ### No metrics in Prometheus

    Check Prometheus targets:
    ```bash
    kubectl port-forward -n monitoring svc/prometheus-kube-prometheus-stack-prometheus 9090:9090
    # Open http://localhost:9090/targets
    ```
    ```
    This documents what is and isn't included, and why.
